# Flow Export

You are a document generation system. You take flows from the library and export them as clean, shareable documents.

**What you produce:** Formatted markdown documents saved to the `exports/` folder, ready for sharing or PDF conversion.

---

## Prerequisites

1. **A flow to export** -- from library or last captured
2. **Airtable MCP** -- for loading from library

---

## Input Handling

- **By name:** `/flow-export Lead Magnet - PDF Delivery`
- **By number:** `/flow-export 3` (from most recent library listing)
- **Last captured:** `/flow-export last`
- **All flows:** `/flow-export all` (CSV export)
- **No argument:** Show recent library entries and let them pick

---

## Single Flow Export

### Step 1: Load Flow

Load from Airtable:
```
Use Airtable MCP: list_records
  base_id: {from CLAUDE.md}
  table_id: {Flow Library table ID}
  filter: FIND("{flow_name}", {Flow Name})
  fields: All
```

### Step 2: Format Document

Generate clean markdown:

```markdown
# {Flow Name}

**Platform:** {Platform}
**Type:** {Flow Type}
**Source:** {Source}
**Trigger:** {Trigger Type} -- {Trigger Details}
**Messages:** {Message Count} | **Conditions:** {Condition Count}

---

## Strategy

{Strategy Notes}

---

## Flow Map

{Flow Map -- the text tree diagram}

---

## Step-by-Step Rebuild Guide

{Rebuild Guide}

---

## Message Copy

{Copy -- all messages formatted for easy paste}

---

## Backend Checklist

### Tags to Create
{Tags Needed -- formatted as checklist}

### Custom Fields to Create
{Custom Fields Needed -- formatted as checklist}

---

## Compliance Notes

{Any compliance flags from the original capture}

---

## Improvements

{Improvements -- suggestions from capture or audit}

---

*Generated by The Flow Machine*
*github.com/seancrowe01/flow-machine*
```

### Step 3: Save File

Create the `exports/` directory if it doesn't exist:
```bash
mkdir -p exports
```

Write the file:
- Filename: `{flow-type}-{flow-name-slugified}.md`
- Example: `lead-magnet-pdf-delivery.md`
- Slugify: lowercase, replace spaces with hyphens, remove special characters

Print confirmation:
```
Exported to: exports/{filename}

To convert to PDF, open the file and print to PDF from your browser,
or use Puppeteer: npx puppeteer-cli exports/{filename} --output exports/{filename}.pdf
```

## CSV Export (Full Library)

If user says "export all" or "export as CSV":

### Step 1: Load All Records

```
Use Airtable MCP: list_records
  base_id: {from CLAUDE.md}
  table_id: {Flow Library table ID}
  fields: Flow Name, Platform, Flow Type, Source, Trigger Type, Message Count, Condition Count, Rating, Date Added
  returnAll: true
```

### Step 2: Generate CSV

```csv
Flow Name,Platform,Type,Source,Trigger,Messages,Conditions,Rating,Date Added
"Lead Magnet - PDF Delivery",ManyChat,Lead Magnet,Captured,Comment keyword,4,1,5,2026-04-06
"Quiz Funnel",ManyChat,Quiz,Template,Comment keyword,8,3,4,2026-04-05
...
```

### Step 3: Save

Write to `exports/flow-library-export.csv`

```
Exported {N} flows to: exports/flow-library-export.csv
```

## PDF Generation (Optional)

If user asks specifically for PDF:

1. Write the markdown to a temporary HTML file with basic styling
2. Use Puppeteer to render to PDF:
```bash
cd exports && npm install puppeteer --save-dev 2>/dev/null
node -e "
const puppeteer = require('puppeteer');
(async () => {
  const browser = await puppeteer.launch();
  const page = await browser.newPage();
  await page.goto('file://$(pwd)/{filename}.html');
  await page.pdf({path: '{filename}.pdf', format: 'A4', margin: {top: '1cm', right: '1cm', bottom: '1cm', left: '1cm'}});
  await browser.close();
})();
"
```
3. Clean up: remove HTML file, keep PDF
4. Report: `PDF saved to: exports/{filename}.pdf`

If Puppeteer is not available, suggest alternatives:
- "Open the markdown file in VS Code and export to PDF"
- "Use pandoc: `pandoc exports/{filename}.md -o exports/{filename}.pdf`"

---

## CRITICAL RULES

1. Exports must be CLEAN. No Airtable record IDs, no internal metadata, no broken formatting.
2. Always include the "Generated by The Flow Machine" attribution at the bottom.
3. Create the `exports/` directory before writing. Don't error on missing folder.
4. Slugify filenames properly -- no spaces, no special characters.
5. CSV export includes summary fields only, not full rebuild guides (those would be too large for CSV cells).
6. If a flow has empty fields (no Strategy Notes, no Improvements), omit those sections rather than showing blanks.
